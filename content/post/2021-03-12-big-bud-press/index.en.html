To summarise a bit:

  • The website has a header and footer that span the entire screen, but the actual content (including the website navigation) is limited to 1180 pixels. This is something I see a lot of lately - I‚Äôm not sure if it‚Äôs a ~trend or if I only noticed because I got a wider monitor, but it‚Äôs really apparent (and awful) now when a store‚Äôs website is full width.
  • The header and footer are the same colour (a dark red, #C13400), and this is used throughout the site - it‚Äôs the colour when you hover over a link in the navigation bar, the colour of the link for the page you‚Äôre currently on, the colour used to outline buttons, for links in text, and more.
  • You might be keen and have looked at Big Bud Press and noticed that everything is turquoise instead of red‚Ä¶ I KNOW! They had a launch and updated the site in the middle of me working on this.   • The navigation bar is also limited in width to 1180 pixels, with a logo on the left and the navigation links pushed to the right. The navigation links are also centered vertically with the logo.
  • The site uses the Cooper BT font, with a 500 weight for regular text, 700 weight for bold, and 900 weight for headings.
  • Their logo and branding is super cute.

The first step is to create a CSS file which will contain all of the rules to style this app. I‚Äôll put this in the www/ directory, with my app file living in the main directory. In order to tell the app to use this CSS file (called style.css), I‚Äôll link it in the app‚Äôs UI. This is pseudo code, but the basic idea!

There isn‚Äôt actually anything in the CSS file, so nothing changes with the app!

I‚Äôll start off with the font and basic colours. Cooper BT actually isn‚Äôt a free font, so I can‚Äôt use it, but I found the Corben font from Google Fonts, which has a similar bubbly 70s vibe. It only comes in two weights - 400 and 700 - and no italics, but good enough for my purposes!

To tell the app to use Corben, first I need to import it into the CSS file:

/* Import font for use */
@import url('https://fonts.googleapis.com/css2?family=Corben:wght@400;700&display=swap');

Next, I‚Äôll define some variables to use in the file. Like in R, you don‚Äôt always want to hardcode values like the font, colour, or sizing, because it gets cumbersome to type them out, and in case you need to change ‚Äôem later‚Ä¶ like if the brand you‚Äôre ripping off changes their website from red to turqouise in the middle of a blog post. I‚Äôll make some variables to store the font I‚Äôm using, the base size, weights, as well as some colours.

:root {
  /* Fonts */
  --text-font-family: "Corben";
  --text-font-base: serif;
  --base-font-size: 18px;
  --base-weight: 400;
  --heavy-weight: 700;

  /* Colours */
  --text-color: #2E3438;
  --background-color: #FFFFFF;
  --primary-color: #00807A;
  --link-color: #00CCC3;
  --white: #FFFFFF;
}

Nothing is actually using these variables yet, though! So I‚Äôll set everything (*, like in SQL) to use the font, size, and text colour I want, referring to variables via something like var(--variable-name). I‚Äôll also set the headers to be a heavier weight, and change the colour of links (and their colour on hover):

/* Set everything to Corben font, with serif as a fallback, and set the colour and size */
* {
  font-family: var(--text-font-family), var(--text-font-base);
  font-size: var(--base-font-size);
  color: var(--text-color);
}

/* Heavier headers */
h1, h2, h3, h4, h5, h6 {
  font-weight: var(--heavy-weight);
}

/* Link colour and hover */
a {
  color: var(--primary-color);
}

a:hover {
  color: var(--link-color);
}

Now the fonts are all cute and bubbly, with chunky headers and a colourful link!

Just a general note - if you‚Äôre just changing the UI of your app, either by editing the actual Shiny code for the UI or the CSS, you don‚Äôt need to stop and restart your app - you can just refresh it to see the changes!

This full width page is really bugging me, so next up I‚Äôll update it so that the page is contained to be 1180 pixels wide, centered in the middle of the page.

Now I‚Äôll tackle the navigation bar. I find this is where I end up focusing most of my energy, but it makes a huge difference. I‚Äôll do the fonts first, making them a little chunkier, converting to uppercase (without actually altering the underlying name of the tab in Shiny!), setting the hover/active colour to be that turqouise (our ‚Äúprimary‚Äù colour in the app), and removing any additional background on the link of the active page.

I know this CSS can look a bit intimidating, especially the selectors - I hope you take some comfort in knowing that I do NOT know what they all do and YES I just copied ‚Äôem from the inspector in Firefox. It works, who cares?

/* Make nav bar links uppercase and heavier weight */
.navbar-default .navbar-nav > li > a {
  text-transform: uppercase;
  font-weight: var(--heavy-weight);
  color: var(--text-color);
}

/* Change to primary colour on hover, make active tab navbar link primary colour, and add no additional background to all */
.navbar-default .navbar-nav > li > a:hover, .navbar-default .navbar-nav > .active > a, .navbar-default .navbar-nav > .active > a:focus, .navbar-default .navbar-nav > .active > a:hover, .navbar-default .navbar-nav > .open > a:hover {
  color: var(--primary-color);
  background-color: transparent;
}

So cute! But it would be cuter with their logo, so I‚Äôll add that in by altering the underlying Shiny code. I‚Äôm making it so that the logo is a link that returns you to first page of the Shiny app, rather than‚Ä¶ doing nothing. The logo also lives in the www/ directory.

In the R code, I load the logo and update the navbarPage code to look a little something like this. The windowTitle argument refers to what you see as the page‚Äôs title in your browser (ie on a tab) - otherwise it would be something like <a href="/" img<... no thanks!

To make the navigation bar fit the logo, I need to update the CSS. I‚Äôll also change the navbar to have a white background, with a thin bottom border in our primary colour and a thick top one (to emulate the header on the website), and adjust the height of the navigation bar links so that they‚Äôre centered with the logo, give a little more space between ‚Äôem, and push them to the right:

  height: 130px; /* Image is 100px + 15px padding top and bottom */

  background-color: var(--white);
  border-bottom: 1px solid var(--primary-color);
  border-top: 30px solid var(--primary-color);
}

  float: right;
}
